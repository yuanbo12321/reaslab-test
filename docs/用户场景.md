## 2. 通过邮箱注册账号

### 2.1 提交注册信息后提示验证邮件已发送

#### 步骤

1. 打开网站上的注册页面，填写用户名、邮箱、密码及确认密码。
2. 提交后，界面应出现成功类提示（如验证邮件已发送）；系统会校验信息并向你的邮箱发送一封验证邮件（内含验证链接）。

#### 在邮箱中完成验证（人工，无 E2E）

1. 在邮箱中点击验证链接，按页面提示完成验证。
2. 验证通过后按第 3 章完成登录；忘记密码见 §3.1「找回密码」。

<a id="toc-3"></a>

## 3. 登录

### 3.1 通过邮箱登录

#### 步骤

1. 打开网站上的登录页面。
2. 输入已验证邮箱与密码并登录。
3. 失败时按页面提示检查密码与邮箱验证状态。

#### 找回密码

**步骤**

1. 在登录相关流程中进入忘记密码页面，输入注册时使用的邮箱并提交。
2. 查收重置密码邮件并按链接打开页面（无对应账号时，界面可能不说明是否存在账号）。
3. 在页面填写邮箱、新密码与确认密码并提交。
4. 返回登录页用新密码登录；本机原登录状态可能被清除。

---

#### 通过邮箱免密码登录（无 E2E）

**步骤**

1. 打开网站上的邮箱登录 / Email sign-in 入口，输入邮箱并请求发送登录链接。
2. 打开邮件中的链接完成登录。
3. 提示「请查收邮件」时，留意收件箱与垃圾箱。

---

#### 通过 GitHub 登录（无 E2E）

**步骤**

1. 在登录页点击 Login with Github（或同等入口）。
2. 在 GitHub 完成授权后返回本站。

## 5. 创建空白项目并使用基础功能

与路线图「创建空白项目并使用基础功能」及 Playwright `test.describe` 标题一致；产品界面中项目类型常为 **数学建模 / Modeling**。**5.1～5.8** 见 `reaslab-test/test/05-blankModeling.test.ts`。

### 5.1 创建空白 Modeling 项目

#### 步骤

1. 登录后在工作台左侧选择 New Project（新建项目）。
2. 填写项目名称，选择项目类型：
   - **数学建模**：获得适合写 Python 与说明文档的默认工程。
3. 创建后进入编辑环境。

---

### 5.2 上传图片

经 ReasLingo 侧上传入口，将 **`reaslab-test/test/data/test_upload.png`** 写入项目（常见路径为 **`chat-uploads/test_upload.png`**），并在文件树或输入区芯片中可见。

---

### 5.3 使用OCR进行AI会话

在默认 Agent 下：等待图片 **OCR Processing** 结束，将模型切为 **Auto**（若界面提供），用**全英文**就图中问题提问，待本轮助理回复结束；界面不应持续出现 **OCR Failed**，且应对图中算术给出正确数字答案（自动化用例校验为 **4**）。

---

### 5.4 切换Paper Copilot进行AI会话

将 Agent 切换为 **Paper Copilot**（名称以线上为准）；通过 **Add Context** 将 **`test_upload.png`** 加入本条消息上下文；模型选 **Auto** 后发送与 §5.3 相同性质的英文题，待回复完成并满足 OCR 与答案校验（与自动化一致）。侧栏 Agent 菜单须含 **Paper Copilot**，否则对应 E2E 失败。

---

### 5.5 邀请他人共同编辑项目

#### 步骤

1. 在项目中打开分享 / Sharing，输入对方邮箱、选择权限（只读/可编辑等以界面为准）并发出邀请。
2. 生成分享链接发给对方；对方登录后打开链接验证即可进入。
3. 长期管理成员与权限：设置 → 协作者。

---

### 5.6 查看项目的修改历史

#### 步骤

1. 打开历史 / History，浏览快照（新建、导入等常会生成快照）。
2. 选中快照查看文件与说明；若有「检出 / 恢复」，可按提示切换工作区。
3. 用差异对比查看与当前或其他版本的差别。

---

### 5.7 项目内搜索关键字

#### 步骤

1. 在项目内打开搜索（入口因布局可能不同）。
2. 输入关键词，筛选后从结果跳转到编辑器。

---

### 5.8 导出项目

#### 步骤

1. 整个项目打包：设置或工具栏下载，导出为压缩包（文件名多为项目名）。
2. 模板附带资料：第 7、9 章模板详情页可下载论文 PDF、附件等（非整包导出入口）。

## 6. 从 Git 导入项目

### 6.1 进入 IDE 并试用 ReasLingo 对话

#### 步骤

1. 在工作台选择 Import Git（导入 Git）。
2. 填写仓库地址（`https://` 或 `git@` 等）；可自定义项目名，默认多为仓库名。
3. 选择类型：数学建模或定理证明。
4. 点击导入；GitHub 私有库按提示授权后由平台拉取。
5. 导入成功后进入项目；定理类需等待环境与依赖就绪。
6. 侧栏打开 ReasLingo，Agent 选默认 Agent、发送 `who are you?` 一条消息，待本轮流式回复结束后再继续。

## 12. 使用 LaTeX 文件

### 12.1 上传本地 LaTeX 并编译预览

与 `reaslab-test/test/12-latex.test.ts` 及夹具 **`reaslab-test/test/data/test_upload.tex`** 对齐（数学建模项目 + Explore **Upload Files** + 可见 **`.cm-content`** 断言 LaTeX + 工具栏 **眼睛 / Toggle TeX Preview** 打开侧栏 + **Compile** + PDF **canvas** 断言）。

#### 步骤

1. 登录后打开 **beta.reaslab.io** 上某一项目的 IDE（URL 形如 `/projects/<uuid>`）。
2. 在左侧文件树上方工具栏使用 **上传**（向上箭头）将本地的 **`.tex` 文件**（若编译依赖其它文件，一并上传）导入项目；在编辑器中打开该 `.tex` 文件（出现对应标签页）。
3. 在编辑器工具栏点击 **眼睛图标**（**Toggle TeX Preview**），打开右侧 TeX / PDF 预览窗；未打开时绿色 **Compile** 不可用或不可见。
4. 在预览区侧栏点击绿色 **Compile** 按钮；编译完成后，预览区应由「Click the compile button to preview PDF」类提示转为 **PDF 预览**（或产品定义的等价编译结果展示）。
5. 需要排查时可打开 **Logs** 等入口查看编译输出；编译失败时预览区或日志中应能看到与 LaTeX 相关的错误信息。

## 13. 项目列表查看及管理

本章宜安排在**其它用户场景与验收之后**执行，作为**收尾**：查看列表与行内能力后，通过 **13.2** 清空测试账号在 **My Projects** 下的自有项目（与定时流水线中 **`13-project-list.test.ts`** 置于末段的目的一致）。

### 13.1 四标签与行内操作

#### 步骤

1. 登录后左侧进入项目。
2. 顶部标签切换：全部、我的项目、分享给我、已归档。
3. **My Projects**：进入后**清空搜索框残留**（`fill("")`，不按关键字搜索），再查看列表；取列表中的项目点击项目名进入。
4. 行内操作（无权限时部分不可用）：Setup（环境与依赖）、Settings、Download（同 §5.8 整包导出）、Copy、Rename、Archive/退出。
5. 已归档中可恢复或永久删除（多有确认）。
6. 多选后出现批量栏时，可按提示归档、退出或恢复/删除。

### 13.2 删除My Projects所有项目

#### 步骤

1. 在 **My Projects**（**不使用搜索框**）：全选 → **Archive** → 按提示确认（若有）→ 切到 **Archived** → 全选 → **Delete** → 按提示确认。
2. 回到 **My Projects**，列表应无数据行（与 `13-project-list.test.ts` 断言一致）。
