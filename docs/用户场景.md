## 1. 免登录在线体验 Playground

2. 在 **Examples** 中选择第一项；**Lean** 编码区应显示示例代码，**Infoview** 区应显示对应内容（如类型、证明目标等）。

## 2. 通过邮箱注册账号

### 步骤

1. 打开网站上的注册页面，填写用户名、邮箱、密码及确认密码。
2. 提交后，系统会校验信息并向你的邮箱发送一封验证邮件（内含验证链接）。
3. 在邮箱中点击验证链接，按页面提示完成验证。
4. 验证通过后按第 3 章完成登录；忘记密码见 3.1 节「找回密码」。

---

<a id="toc-3"></a>

## 3. 登录

### 3.1 通过邮箱登录

#### 步骤

1. 打开网站上的登录页面。
2. 输入已验证邮箱与密码并登录。
3. 失败时按页面提示检查密码与邮箱验证状态。

#### 找回密码

**步骤**

1. 在登录相关流程中进入忘记密码页面，输入注册时使用的邮箱并提交。
2. 查收重置密码邮件并按链接打开页面（无对应账号时，界面可能不说明是否存在账号）。
3. 在页面填写邮箱、新密码与确认密码并提交。
4. 返回登录页用新密码登录；本机原登录状态可能被清除。

---

### 3.2 通过邮箱免密码登录

#### 步骤

1. 打开网站上的邮箱登录 / Email sign-in 入口，输入邮箱并请求发送登录链接。
2. 打开邮件中的链接完成登录。
3. 提示「请查收邮件」时，留意收件箱与垃圾箱。

---

### 3.3 通过 GitHub 登录

#### 步骤

1. 在登录页点击 Login with Github（或同等入口）。
2. 在 GitHub 完成授权后返回本站。

## 5. 创建空白 Modeling 项目并使用基础功能

### 5.1 创建空白 Modeling 项目

#### 步骤

1. 登录后在工作台左侧选择 New Project（新建项目）。
2. 填写项目名称，选择项目类型：
   - **数学建模**：获得适合写 Python 与说明文档的默认工程。
3. 创建后进入编辑环境。

---

### 5.2 使用 AI 智能体

在 ReasLingo 打开 Agent 菜单（名称以线上为准），选中后在对话区输入问题。

**测试内容（验收 / 自动化）**

1. 将仓库内 **`reaslab-test/test/data/test_upload.png`** 上传至当前项目（经 ReasLingo 侧「为对话上传文件」入口，使文件进入工程树，供后续引用）。
2. 将 Agent 切换为 **Paper Copilot**（或名称含 paper copilot；以线上为准）。
3. 点击输入区上方的 **Add Context** 按键，在浮层中搜索并选中 **`test_upload.png`**，将其加入本条消息上下文。
4. 发送消息，使 AI 根据 **`test_upload.png`** 图中问题作出回复（流式输出结束即视为本轮回复完成）。



---

### 5.3 邀请他人一起编辑项目

#### 步骤

1. 在项目中打开分享 / Sharing，输入对方邮箱、选择权限（只读/可编辑等以界面为准）并发出邀请。
2. 生成分享链接发给对方；对方登录后打开链接验证即可进入。
3. 长期管理成员与权限：设置 → 协作者。

---

### 5.4 查看项目的修改历史

#### 步骤

1. 打开历史 / History，浏览快照（新建、导入等常会生成快照）。
2. 选中快照查看文件与说明；若有「检出 / 恢复」，可按提示切换工作区。
3. 用差异对比查看与当前或其他版本的差别。

---

### 5.5 项目内搜索关键字

#### 步骤

1. 在项目内打开搜索（入口因布局可能不同）。
2. 输入关键词，筛选后从结果跳转到编辑器。

---

### 5.6 导出项目

#### 步骤

1. 整个项目打包：设置或工具栏下载，导出为压缩包（文件名多为项目名）。
2. 模板附带资料：第 7、9 章模板详情页可下载论文 PDF、附件等（非整包导出入口）。

---

<a id="toc-6"></a>

## 6. 从 Git 导入项目

### 步骤

1. 在工作台选择 Import Git（导入 Git）。
2. 填写仓库地址（`https://` 或 `git@` 等）；可自定义项目名，默认多为仓库名。
3. 选择类型：数学建模或定理证明。
4. 点击导入；GitHub 私有库按提示授权后由平台拉取。
5. 导入成功后进入项目；定理类需等待环境与依赖就绪。
6. 侧栏打开 ReasLingo，Agent选默认Agent、发"who are you?"一条消息，待本轮流式回复结束后再继续。

## 10. 查看项目列表并管理项目

### 步骤

1. 登录后左侧进入项目。
2. 顶部标签切换：全部、我的项目、分享给我、已归档。
3. 搜索框按名称筛选；点击项目名进入。
4. 行内操作（无权限时部分不可用）：Setup（环境与依赖）、Settings、Download（同 5.6 整包导出）、Copy、Rename、Archive/退出。
5. 已归档中可恢复或永久删除（多有确认）。
6. 多选后出现批量栏时，可按提示归档、退出或恢复/删除。

## 11. 向 ReasLab 反馈意见

### 步骤

1. 登录后进入工作台首页。
2. 左侧主导航底部点 Feedback，在新标签页打开 GitHub 新建 Issue。
3. 填写标题与正文（建议含复现步骤、期望与实际现象；可附截图或链接）并提交。

## 12. 页脚导航链接

### 步骤

1. 在工作台首页（或同等营销首页）滚动至页脚，点击 **Templates**（如 Optimization Modeling）、**Resources**、**About** 等链接。
